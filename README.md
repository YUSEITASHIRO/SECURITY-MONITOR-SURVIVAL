# 🎥 SECURITY MONITOR SURVIVAL

> **監視カメラを通じて恐怖と対峙する、サバイバルホラーゲーム**

---

## 📖 概要

あなたは深夜の警備員。複数台の監視カメラを切り替えながら、施設内を徘徊する**謎の怪物**の動きを常に視界に収め続けなければなりません。

あなたがモンスターから目を離した瞬間——それは、あなたの命が終わる瞬間です。

---

## 🎮 ゲームルール

- **ターン制**: ゲームは複数のターンで構成されています（全5ターン）
- **ライフ制**: 初期ライフは3。モンスターを一定時間（3秒）視界から外すと1失います
- **監視ルール**: モンスターをカメラの視野（FOV）内に収め続けることが目標
- **クリア条件**: 全ターンを生き延びる

---

## ⌨️ 操作方法

| キー | 操作 |
|---|---|
| `0` 〜 `9` | 監視カメラの切り替え（CH0〜CH9） |
| `←` `→` 矢印キー | カメラの向きを左右に回転 |
| `Y` | 監視フェーズ開始 |
| `ENTER` | メニュー確定 |

---

## 🤖 モンスターの特徴

モンスターは単純に動き回るだけではありません。**ターンが進むにつれてより賢く・危険**になっていきます。

- 📷 **カメラに映ったことを察知する** → 最も近い交差点へ超高速でダッシュし、死角へ逃げ込む
- 🔌 **カメラ故障を利用する** → 故障中のカメラを見ているとバレた瞬間、猛スピードで接近してくる
- 🏃 **視認されるほど行動が速くなる** → 監視が続くほどモンスターの反応が素早くなる

---

## 📷 カメラ故障イベント

ターン2以降、ごく稀に監視カメラが突然故障します。

- 故障したカメラは**ホワイトノイズ（砂嵐）**で映像が見えなくなります
- UI上では対象カメラのボタンが **`[x] ERR`（赤色）** で表示されます
- ミニマップ上のカメラアイコンも **赤色の `X`** になります
- **故障中はそのカメラでモンスターを視認できません**（要注意！）

## 🛠 技術情報（開発者向け）

### 動作環境
- Windows 10 / 11 (x64)
- .NET 9.0ランタイム（Self-containedビルドの場合は不要）

### 使用技術
- **言語**: C# / .NET 9.0
- **グラフィックスライブラリ**: [Raylib-cs](https://github.com/chrisdill/raylib-cs) v7.0.2
- **3D描画**: Raylib の Mode3D を使用したカメラ視点レンダリング
- **レンダーテクスチャ**: 監視カメラ映像のリアルタイムレンダリング

### ビルド方法

```bash
# Debug ビルド
dotnet build HorrorGame.csproj

# Release（Windows x64 実行ファイル生成）
dotnet publish HorrorGame.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true -o ./release
```

### ファイル構成

```
LLMwork/
├── Program.cs          # メインゲームロジック（全コード）
├── HorrorGame.csproj   # プロジェクト設定
├── README.md           # このファイル
└── release/
    └── HorrorGame.exe  # 実行ファイル（ビルド後に生成）
```
